# LeetCode刷题技术要点与学习计划

## 常用技术要点

### 1. 数据结构基础
- **数组与字符串**：双指针、滑动窗口、前缀和、字符串匹配
- **链表**：快慢指针、虚拟头节点、翻转操作
- **栈与队列**：单调栈、双端队列、优先队列
- **树**：前/中/后序遍历、层序遍历、路径问题
- **图**：BFS、DFS、拓扑排序、最短路径
- **哈希表**：计数、存储状态、快速查找
- **堆**：TopK问题、数据流中位数

### 2. 算法思想
- **排序算法**：快速排序、归并排序、计数排序、桶排序
- **二分查找**：有序数组查找、旋转数组查找、二分答案
- **双指针技巧**：快慢指针、左右指针、滑动窗口
- **贪心算法**：区间问题、分配问题
- **分治算法**：大问题分解为小问题
- **回溯算法**：组合、排列、子集、剪枝技巧
- **动态规划**：状态定义、转移方程、维度压缩
- **位运算**：异或、与或非、位移操作

### 3. 高级数据结构
- **线段树/树状数组**：区间查询与修改
- **并查集**：连通性问题
- **字典树(Trie)**：前缀匹配、单词查找
- **跳表**：快速查找
- **红黑树/AVL树**：自平衡树
- **LRU/LFU缓存**：缓存淘汰策略

### 4. 高级算法
- **KMP算法**：字符串匹配
- **Dijkstra算法**：最短路径
- **拓扑排序**：有向无环图的节点排序
- **Tarjan算法**：寻找强连通分量
- **并行算法**：多线程解决问题
- **数学算法**：数论、组合数学

## 刷题技巧与方法

### 1. 分类刷题
按照数据结构和算法类型分类刷题，同类型题目一起做，有助于形成解题模式

### 2. 重复刷题
- 第一遍：尝试独立思考，如果30分钟没思路则查看题解
- 第二遍：1周后重做，加深理解
- 第三遍：1个月后重做，巩固记忆

### 3. 做题方法
- 理解题目，确认边界条件
- 分析并选择合适的数据结构和算法
- 先写出正确解法，再考虑优化
- 写代码前思考测试用例
- 代码实现后进行自测

### 4. 总结归纳
- 每周对做过的题目进行分类总结
- 整理常见的解题模板和技巧
- 记录自己容易出错的地方
